## 核心比喻：共同工作空間

> **AInTandem 不是「沙盒」——它是為 AI agents 打造的共同工作空間。**

在這個空間裡：
- 每個 **Sandbox** 是一個獨立的工作間包廂
- 每個 **Agent** 都有自己的辦公桌、工具和狀態
- 它們彼此不干擾，但可以被有序地連接起來
- **這不是讓 AI「試試看」，而是讓 AI 真正坐下來上班。**

這是從「沙盒」（測試、臨時、隔離）到「**共同工作空間**」（生產、協作、持久）的根本轉變。

---

## 執行摘要

AInTandem 是一個**安全、本地的人機協作共同工作空間**。每個沙盒提供一個安全的工作區，AI agents 可以在這裡與人類並肩工作來完成任務——**從網頁自動化到軟體開發**。

我們的終極目標是成為**一個能夠八成獨立運作的軟體工程團隊**。

不是輔助工具。不只是 AI coding。

而是一群 AI agents，能夠：
- 被交付長時間運行的任務
- 在環境內協作
- 產出可用的成果

**當你只有一個人，但你需要一個工程團隊的時候，AInTandem 可以補上那八成。**

## 我們解決的核心問題

### 真正的問題：信任

很多人談 AI「能力」。但我們專注於一個更實際的問題：

> **我敢不敢把任務交給它，然後離開？**

不是離開 5 分鐘。是離開 5 小時、一天、甚至一個週末。

- 它會不會亂刪檔案？
- 會不會跑到不該跑的 port？
- 會不會彼此踩到環境？
- 出事了，我能不能回來收拾？

**AInTandem 第一階段做的不是「更聰明的 AI」——而是建立一個讓使用者心理上真的能放得下的環境。**

### YOLO 模式：Agent 自由運行，風險受控

我們說 AInTandem 為 AI agents 啟用了 **YOLO mode**。但 YOLO 不是亂來，而是：

- Agents 可以長時間自主行動
- 可以反覆嘗試、失敗、修正
- 可以同時跑多個任務
- **而我不用盯著它**

前提是：環境本身是安全的、可控的、可回收的。

## 核心價值主張

### 主要價值主張

1. **安全的本地 AI 執行**
   - 所有 AI agent 活動都在隔離的本地沙盒中進行
   - 你的資料從不離開你的機器
   - 完全控制和可見性於 agent 行動
   - 雲端 AI 自動化的安全替代方案

2. **通用任務自動化**
   - 透過 xvfb + Chromium 進行網頁自動化（無頭瀏覽器）
   - 多種語言的程式碼執行（Node.js、Python 等）
   - 檔案操作和資料處理
   - 透過 MCP（Model Context Protocol）整合

3. **零複雜度設定**
   - 一鍵安裝的桌面應用程式
   - 預先配置的沙盒，包含所有必要工具
   - 不需要 Docker 專業知識
   - 基於網頁的 UI，易於互動

4. **人機協作**
   - 透過即時 shell/瀏覽器存取監控你的 AI agent
   - 必要時介入和引導
   - 從 agent 行動中學習
   - 透過透明度建立信任

### 我們不專注的（戰略決定）

> **Context Management** — 不是核心差異化點
>
> 隨著 LLM、Skills/Agents 框架和 MCP（Model Context Protocol）的進步，我們認為：
> - 外部系統可以透過 MCP 連接
> - 許多專案可以透過基於 Markdown 的工作流程管理
> - 知識管理最好由專門的工具處理

#### 關鍵差異化：什麼在哪裡執行

| 活動 | 位置 | 原因 |
|----------|----------|-----|
| **AI Agent 執行** | ❌ 從不在雲端 | 資料隱私、安全性、合規 |
| **程式碼處理** | ❌ 從不在雲端 | 你的程式碼屬於你 |
| **檔案儲存** | ❅ 從不在雲端 | 完全控制、無廠商鎖定 |
| **網頁瀏覽** | ❅ 從不在雲端 | xvfb+Chromium 本地執行 |
| **通訊隧道** | ✅ 雲端服務 | 從任何地方安全存取 |
| **身份驗證** | ✅ 雲端服務 | SSO、存取管理 |
| **更新與修補** | ✅ 雲端服務 | 無縫維護 |

**我們的雲端是橋樑，不是工廠。**

你的 AI agents 在你的共同工作空間本地運作。雲端服務只提供安全隧道，讓你可以從任何地方存取它們——不需要設定 SSH、配置 VPN 或管理 DNS。

## 競爭優勢

### 共同工作空間的差異化

| 傳統 AI 工具 | AInTandem 共同工作空間 |
|--------------|---------------------------|
| **助理** — AI 回應你的提示 | **同事** — AI 與你並肩工作 |
| **單次** — 在單一會話中完成任務 | **持久** — Agents 有持續的工作空間 |
| **短暫** — 會話之間無記憶 | **累積** — 工作隨時間增長 |
| **黑盒** — 看不到發生什麼 | **透明** — 即時監控 agents 工作情況 |
| **依賴雲端** — 資料離開你的機器 | **本地優先** — 一切留在你的機器 |

### 核心優勢（共同工作空間比喻）

| 優勢 | 比喻 |
|------|------|
| **隱私優先** | 你自己的私人辦公室 — 沒有其他人能存取 |
| **內建瀏覽器** | 預先配備的工作站 — xvfb + Chromium 就緒 |
| **視覺透明** | 玻璃牆 — 清楚看到你的 agents 在做什麼 |
| **開發者+非開發者** | 對所有專業人士開放 — 不限於程式設計師 |
| **自架** | 你擁有這棟建築 — 完全控制、無房東 |
| **MCP 原生** | 通用電源插座 — 插入任何工具 |

### vs. 雲端 AI 自動化工具

| 層面 | AInTandem | 雲端工具 (Zapier, Make) |
|------|-----------|---------------------------|

| **瀏覽器存取** | ✅ 內建 | ⚠️ 有限或昂貴 |
| **自訂程式碼** | ✅ 完整存取 | ⚠️ 受限 |
| **長時間任務** | ✅ 離開後回來 | ⚠️ 超時限制 |

### vs. 開發者工具

| 層面 | AInTandem | Devbox, Daytona, Codespaces |
|------|-----------|----------------------------|
| **主要比喻** | ✅ 共同工作空間 | ⚠️ IDE/環境 |

| **網頁自動化** | ✅ 內建 | ❌ 不為此設計 |
| **多 Agent** | ✅ Agents 協作 | ⚠️ 單人焦點 |
| **本地優先** | ✅ 是 | ⚠️ 依賴雲端 |

## 產品願景聲明

> **"AInTandem 是一個人類與 AI agents 並肩工作的共同工作空間。當你只有一個人，但你需要一個團隊時，我們補上那八成。"**

### 宣言

| 原則 | 含義 |
|------|------|
| **不取代，而是增強** | AI agents 不取代人類 — 它們與你並肩工作 |
| **你的資料，你的控制** | 一切在本地執行。無雲端依賴，無資料離開你的機器 |
| **透過透明度建立信任** | 清楚看到你的 AI agents 在做什麼。即時監控它們的工作 |
| **長時間自主運作** | 交付任務，離開，回來時看成果 |
| **安全實驗** | 隔離的工作空間意味著 agents 可以探索而不破壞你的系統 |
| **心理安全** | 一個你真正可以「放手」的環境 |

### 「80% 獨立團隊」的意義

| 人類做 (20%) | AI Agents 做 (80%) |
|-----------------|-------------------|
| 定義目標和方向 | 執行實作 |
| 審查和批准 | 撰寫和測試程式 |
| 做出最終決策 | 執行重複性任務 |
| 處理邊緣情況 | 處理資料 |
| 提供創意 | 自動化工作流程 |
| **架構** | **建構** |

這不是關於完美。而是**夠好、可靠、可累積**。
