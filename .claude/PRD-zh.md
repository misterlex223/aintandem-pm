## 1. 執行摘要

AInTandem 是一個統一的 AI 輔助開發平台，使開發者能夠建立、管理和協調容器化開發環境與 AI agent 整合。

**目標發布**: 2025 Q2
**關鍵目標**:
1. 建立清晰的文件結構
2. 減少首次沙盒時間到 < 5 分鐘
3. 建立社群基礎（1,000+ 用戶）

## 6. 技術考量

### 架構決策

| 決策 | 理由 |
|------|------|
| **自架優先** | 隱私擔憂、開發者偏好 |
| **MCP 用於整合** | 利用生態系統、避免重新發明 |
| **桌面應用焦點** | 比雲端託管更低的進入障礙 |
| **Context 系統去強調** | LLM 原生改進降低差異化 |

### 技術堆疊

| 組件 | 技術 | 說明 |
|-----------|------------|-------|
| 前端 | React + Vite + Tailwind + Shadcn UI | 部署到 Cloudflare |
| 後端 | Node.js (Express/Fastify) | ce-orchestrator |
| 桌面 | Electron | kai-desktop |
| 容器 | Docker | 必要相依性 |
| SDK | TypeScript | orchestrator-sdk |

## 7. 相依性與風險

### 相依性

| 項目 | 類型 | 影響 |
|------|------|------|
| Docker Desktop | 外部 | 高 - 用戶必須擁有它 |
| Cloudflare Pages | 外部 | 中 - 前端託管 |
| Claude/Qwen APIs | 外部 | 高 - AI 功能 |

### 風險

| 風險 | 機率 | 影響 | 緩解 |
|------|------|------|------|
| Docker 要求阻礙用戶 | 高 | 高 | 桌面安裝程式搭配 Docker |
| 低採用 | 中 | 關鍵 | 內容行銷、社群建立 |
| 無法變現 | 中 | 高 | 從第一天就專注於企業 |
